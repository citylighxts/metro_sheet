## Penjelasan Kode

MetroSheet adalah aplikasi Flutter untuk scan, nyimpen, dan baca not musik. Alurnya simpel yaitu kita scan lembaran not pakai kamera, simpan ke HP, lalu bisa dibuka lagi kapan aja.

App ini pakai dua database. SQLite dipakai untuk penyimpanan lokal di device supaya tetap bisa dipakai offline. Firestore dipakai buat backup dan sinkronisasi ke cloud. Untuk login, pakai Firebase Auth.

---

### Struktur Folder `lib/`

Struktur projectnya dipisah biar rapi dan gampang dicari.

- `models/` berisi bentuk data yang dipakai di app  
- `services/` isinya fungsi yang berhubungan dengan hal luar seperti Firebase, kamera, atau notifikasi  
- `viewmodels/` ini bagian logika utama, termasuk state management  
- `views/` adalah halaman UI  
- `widgets/` komponen kecil yang bisa dipakai ulang di banyak tempat  

---

### `main.dart`

Ini adalah titik awal aplikasi. Di sini Firebase diinisialisasi dulu, lalu app dijalankan dengan `ProviderScope` karena pakai Riverpod.

Setelah itu langsung masuk ke `AuthWrapper` untuk menentukan user sudah login atau belum.

---

### Models

- `SheetMusic`  
  Menyimpan data satu lembar not, seperti judul, komposer, path gambar, dan tanggal. Data ini bisa diubah ke format Map supaya bisa disimpan ke SQLite.

- `UserProfile`  
  Menyimpan data user seperti uid, email, dan nama. Bisa dipakai untuk Firestore maupun SQLite.

---

### Services

#### `auth_service.dart`

Bagian ini jadi perantara ke Firebase Auth. Semua proses seperti register, login, dan logout ada di sini. Error dari Firebase juga sudah diterjemahkan supaya lebih gampang dipahami user.

#### `database_service.dart`

Service ini handle dua database sekaligus.

- SQLite untuk penyimpanan lokal  
- Firestore untuk backup ke cloud  

Cara kerjanya, setiap kali tambah sheet music, data akan disimpan dulu ke SQLite. Setelah itu baru dicoba sync ke Firestore. Kalau sync gagal, app tetap jalan tanpa error fatal.

#### `device_service.dart`

Bagian ini berhubungan langsung dengan fitur device.

- Scan dokumen pakai kamera  
- Simpan hasil scan ke storage lokal  
- Kirim notifikasi kalau scan berhasil disimpan  

---

### ViewModels

State management di sini pakai Riverpod, jadi logic dipisah dari UI.

#### `auth_viewmodel.dart`

- `RegisterNotifier`  
  Saat register, user dibuat di Firebase, lalu langsung logout. Setelah itu datanya disimpan ke SQLite lokal.

- `ensureLocalUserProvider`  
  Fungsinya memastikan data user selalu ada di SQLite setiap kali login.

#### `home_viewmodel.dart`

Handle semua operasi terkait sheet music seperti load, tambah, update, dan hapus.

#### `sheet_viewer_viewmodel.dart`

State sederhana untuk fitur viewer.

- `isPlaying` untuk status autoscroll  
- `autoscrollEnabled` untuk nyalain atau matiin auto scroll  

Kalau play diaktifkan, autoscroll juga ikut aktif.

---

### Views

#### `auth_wrapper.dart`

Ini semacam gerbang utama. Dia memantau status login dari Firebase.

- Kalau sudah login, langsung ke `HomeView`  
- Kalau belum, diarahkan ke `LoginView` atau `RegisterView`  

#### `home_view.dart`

Halaman utama dengan dua tab.

- Library untuk daftar semua sheet music  
- Settings untuk info akun dan logout  

Di bagian bawah ada tombol scan.

Saat tombol scan ditekan, app akan minta izin kamera, lalu buka scanner. Setelah scan selesai, user diminta isi judul dan komposer sebelum disimpan.

#### `sheet_viewer_view.dart`

Halaman untuk membaca not musik.

Gambar hasil scan ditampilkan dan bisa di-zoom atau digeser. Di bagian bawah ada kontrol autoscroll dengan slider untuk mengatur kecepatan.

---

## Alur Utama

Saat app dibuka, `AuthWrapper` akan cek apakah user sudah login.

Kalau belum login, user masuk ke halaman login atau register. Kalau sudah login, langsung masuk ke Home.

Di Home, user bisa scan not musik baru. Setelah scan dan isi metadata, data disimpan ke SQLite dan dicoba sync ke Firestore.

Semua sheet yang sudah disimpan akan muncul di Library. Kalau salah satu dipilih, akan terbuka di viewer dan bisa dibaca dengan scroll manual atau autoscroll.
